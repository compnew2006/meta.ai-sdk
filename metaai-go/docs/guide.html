The Problem This Solves

A captured, test-locked transport

Built from live browser captures (June 2026). The wire format was reverse-engineered by inspecting real WebSocket frames.

  • Template mode: Captured browser frames with byte-level substitution
  • Correct transport: gorilla/websocket with exact browser headers
  • Header case fix: rupload.meta.ai is header-case sensitive
  • Image attachment: Reverse-engineered the attachment protobuf field

Installation & Setup

Install

go get github.com/smart-studio/metaai-go

Get Cookies

  1. Log into meta.ai
  2. DevTools (F12) → Application → Cookies → https://meta.ai
  3. Copy datr and ecto_1_sess
  4. For token, run in Console: document.documentElement.outerHTML.match(/ecto1:[A-Za-z0-9_-]{20,}/)[0]

Configure

export META_AI_DATR=your_datr
export META_AI_ECTO_1_SESS=your_session
export META_AI_ACCESS_TOKEN=ecto1:your_token
Note: The ecto1 token expires. Re-extract if uploads fail. Chat (WebSocket) is more tolerant.

Chat

package main

import ("context";"fmt";"log";"github.com/smart-studio/metaai-go")

func main() {
    client, _ := metaai.NewClient()
    ctx := context.Background()

    reply, _ := client.Chat(ctx, "What is 2+2?", nil)
    fmt.Println(reply) // "2 + 2 = 4"

    // With options
    opts := &metaai.ChatOptions{NewConversation:true, Thinking:bp(true)}
    reply, _ = client.Chat(ctx, "Explain Go channels", opts)
}

Image Upload & Analysis

// Upload
res, _ := client.UploadImage(ctx, "photo.jpg")
fmt.Printf("mediaID: %s\n", res.MediaID)

// Upload + Analyze
analysis, _ := client.AnalyzeImage(ctx, "photo.jpg", "", "What's in this?", nil)
fmt.Println(analysis)
Verified: Uploaded green image → AI said "Green". The AI sees the actual uploaded image.

Architecture

Template Mode

Instead of a hand-rolled encoder (within 2 bytes but server rejects it), the SDK uses captured browser frames:

  1. Working frame embedded via //go:embed
  2. Substitute: text, conversation ID, request ID, (optionally) media attachment
  3. Preserve all other bytes verbatim
  4. Frame header preserves server's body+2 length quirk

Key Files

FilePurpose
chat.goWebSocket chat (template mode)

internal/transport/gorilla/websocket dialer
